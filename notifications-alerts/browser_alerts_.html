Understanding Browser Alerts

Browser alerts are native dialog boxes that interrupt the user's interaction with a webpage. They are part of the JavaScript's built-in functionality and come in three main types: alert, confirm, and prompt. These dialogs are modal, meaning they block interaction with the rest of the page until they are dismissed.

Key Features of Browser Alerts:

  • Modal dialogs that block interaction with the page
  • Cannot be styled with CSS (browser-controlled appearance)
  • Three distinct types with different user interaction patterns
  • Synchronous execution (JavaScript pauses until user responds)
  • Accessible via keyboard navigation

Types of Browser Alerts:

  1. Alert: Simple notification with an OK button. Used to display information that requires acknowledgment.
  2. Confirm: Question dialog with OK and Cancel buttons. Returns boolean value based on user's choice.
  3. Prompt: Input dialog with text field, OK and Cancel buttons. Returns user input as string or null.

Common Use Cases:

  • Displaying critical notifications or warnings
  • Confirming destructive actions (e.g., delete operations)
  • Collecting simple user input without form elements
  • Interrupting user flow for important messages

// Basic syntax examples:

alert("This is an alert message");

const userConfirmed = confirm("Do you want to proceed?");

const userInput = prompt("Please enter your name:", "Default value");

Test Scenarios & Test Cases

  • Scenario 1: Basic Alert Dialog
    • Verify alert displays with correct message
    • Verify page interaction is blocked during alert
    • Verify script execution continues after alert is dismissed
  • Scenario 2: Confirm Dialog Interaction
    • Verify confirm dialog returns true when OK is clicked
    • Verify confirm dialog returns false when Cancel is clicked
    • Verify conditional logic based on confirm result
  • Scenario 3: Prompt Dialog Data Collection
    • Verify prompt returns user input when OK is clicked
    • Verify prompt returns null when Cancel is clicked
    • Verify default value is displayed in prompt input field
  • Scenario 4: Timed Alert Behavior
    • Verify alert appears after specified timeout
    • Verify page remains interactive until alert appears
    • Test cancellation of timed alert before it appears
  • Scenario 5: Nested Alert Sequence
    • Verify alerts appear in correct sequence
    • Test data flow between sequential alerts
    • Verify all alerts must be dismissed to continue
  • Scenario 6: Customized Prompt Validation
    • Test input validation with custom prompt
    • Verify re-prompting on invalid input
    • Test handling of empty input values
