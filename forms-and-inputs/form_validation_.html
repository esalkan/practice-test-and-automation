Understanding Form Validation

Form validation is a crucial aspect of web development that ensures user input meets specified criteria before submission. It enhances user experience and data integrity. This example demonstrates validation for username, email, password, and age fields.

Key Features of This Form Validation:

  • Real-time feedback on user input
  • Prevention of invalid data submission
  • Custom error messages for specific validation rules
  • Support for text, email, password, and number input types
  • Optional field validation (age)

Validation Rules:

  • Username: 3-20 characters
  • Email: Valid email format
  • Password: At least 8 characters
  • Age: Optional, between 18 and 100 (if provided)

Test Scenarios & Test Cases

  • Scenario 1: Username Validation
    • Test valid username (3-20 characters)
    • Test username too short (less than 3 characters)
    • Test username too long (more than 20 characters)
  • Scenario 2: Email Validation
    • Test valid email format
    • Test invalid email format
  • Scenario 3: Password Validation
    • Test valid password (at least 8 characters)
    • Test password too short (less than 8 characters)
  • Scenario 4: Age Validation
    • Test valid age (18-100)
    • Test age below minimum (less than 18)
    • Test age above maximum (more than 100)
  • Scenario 5: Form Submission
    • Test successful form submission with all valid inputs
    • Test form submission with one or more invalid inputs
  • Scenario 6: Real-time Validation
    • Test error messages appear as user types invalid input
    • Test error messages disappear when user corrects input
  • Scenario 7: Edge Cases
    • Test with maximum allowed characters for each field
    • Test with special characters in username and password
    • Test with whitespace at start and end of inputs
